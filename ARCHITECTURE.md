# Zee — アーキテクチャ概要

> 本ドキュメントは**高レベルの設計概要**です。罠の具体的な作動条件・閾値・回避に直結するルールは含みません（それらは申請者限定で開示します）。

## 位置づけ

Zeeは**侵入後防御（post-intrusion defense）**のレイヤーです。
ファイアウォール・EDR・パッチといった入口防御を置き換えるものではなく、それらをすり抜けて内部に侵入した相手に対して作動します。

設計思想は **「壊れないように作る」から「壊される前提で設計する」へ**。
侵入そのものを防ぐのではなく、**侵入が成果につながらない状態**を作ります。

## 全体の流れ

```
              ┌─────────────────────────────────────────────┐
              │  正規の業務トラフィック（巻き込まない）        │
              └─────────────────────────────────────────────┘
                              │
   侵入した攻撃者 ──▶ ①ふるまい検知 ──▶ ②番人（安全装置）──▶ 引き込み
                                            │（誤作動を確定的に抑止）
                                            ▼
                          ┌─────────────────────────────┐
                          │  罠環境                       │
                          │   ③偽データ汚染               │
                          │   ④偽内部ネットワーク         │
                          └─────────────────────────────┘
                                            │
                                            ▼
                                   ⑤長期相関（低速APT検知）
```

## 各層の役割

| 層 | 役割 | 公開範囲 |
|---|---|---|
| ① ふるまい検知 | 全プロセスの正常なふるまいを学習し、逸脱を検知する一般的な異常検知 | 仕組みは公開 |
| ② 番人（安全装置） | 検知と引き込みの間に立ち、正規プロセス・社員を巻き込まないよう確定的に判定 | 仕組みは公開／作動条件は限定 |
| ③ 偽データ汚染 | 盗まれるデータを目印つきの偽物にすり替え、使用時に発信元を露見させる | 仕組みは公開／生成・目印ルールは限定 |
| ④ 偽内部ネットワーク | 本物そっくりの偽環境で探索のコストと時間を引き上げる | 仕組みは公開／構成ルールは限定 |
| ⑤ 長期相関 | 数ヶ月スパンの潜伏型攻撃を時間をまたいで結びつけて検知 | 仕組みは公開／判定閾値は限定 |

## 安全性の原則（誤作動防止）

侵入後防御の最大のリスクは、**正規のプロセスや社員を誤って罠に引き込むこと**です。Zeeは次の原則で設計しています。

- 検知と引き込みの間に**二重の安全装置**を置く。
- 「誤作動ゼロ」と「重大な攻撃の検知」という相反する要件を、1つの仕組みに詰め込まず**役割を分けて**両立させる。
- 判定はAIの裁量ではなく、**確定的なルール**（許可リスト・多重シグナル）で行う。

## プライバシーの原則

- 生のIPアドレス・個人を特定できる情報は、ログのどの欄にも平文で残さない（保存直前に秘匿・ハッシュ化）。
- 蓄積データは期限を過ぎると自動的に削除する。
- 相関は「個人」ではなく「セッション（行動の主体）」を軸に行うため、正規ユーザーは分析対象に入らない。

## 依存と配布

- Python の標準機能のみで動作（依存を増やさない＝攻撃面を増やさない）。
- 公開：仕組みの全体像・本ドキュメント・論文。
- 申請者限定：罠の具体的な作動ルール・条件・閾値（法人番号等で本人確認／マイナンバーは収集しない）。

## 設計の数学的根拠

罠の核となる決定不能性は、アフィンCollatz力学の素数生存性に基づきます。
