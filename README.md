# Zee

**侵入は、もう100%は防げない。だからZeeは、侵入された“後”を守る。**

Zeeは、攻撃者が内部に入ってきた後に作動する、無料・オープンソースの**侵入後防御レイヤー**です。
入口の防御（ファイアウォール・EDR・パッチ）を置き換えるものではなく、それらをすり抜けた相手に対して **「盗ませて・無力化し・暴く」** ことで、侵入が成果につながらない状態を作ります。

> ⚠️ 本リポジトリは概念実証（PoC）です。本番導入の前に、ご自身の環境での検証を行ってください。詳しくは [免責](#免責) を参照。

## なぜ今か — 機械速度の攻撃

脆弱性の「発見」と「武器化」が、安く・速く・大量になりました。以下はいずれも**開発元であるAnthropic自身が公表した数字**です。

- 旧世代モデルが数百回中わずか2回しか攻撃コードに仕立てられなかったブラウザ（Firefox の JavaScript エンジン）の脆弱性を、新世代モデルは **181回** 動作する攻撃コードに変えた。
- **27年間**、専門家監査と自動テストを生き延びた OpenBSD の欠陥を、**約1,000回の自動試行・総額2万ドル未満**で発見した。
- 横断的な調査では **23,000件超** の脆弱性が見つかり、約 **50億台** の機器に影響する暗号ライブラリの致命的欠陥（CVE-2026-5194）もこの流れで発見されている。

出典：Anthropic Red Team（[red.anthropic.com](https://red.anthropic.com/)）／ Project Glasswing

> これらは**攻撃・調査側の能力**を示す数字です。Zeeが扱うのは**侵入された後の防御**であり、レイヤーが異なります。両者を直接比較することはできません。

発見と武器化がここまで安く・速く・大量になった以上、人手の修正がすべての穴を塞ぎきる前に、攻撃側は次の入口を見つけます。
セキュリティの発想は、「壊れないように作る」から **「壊される前提で設計する」** へ。Zeeはその“後”を引き受けます。

## 仕組み — 5つの層

| 層 | はたらき |
|---|---|
| ① ふるまい検知 | 全プロセスの“いつもの動き”を学習し、逸脱を検知。未知の手口にも反応 |
| ② 引き込み（番人つき） | 怪しい動きを本物そっくりの罠へ誘導。正規の社員・プロセスを巻き込まない設計 |
| ③ 偽データ汚染 | 盗まれるデータを目印つきの偽物にすり替え。使われた瞬間に攻撃者インフラが露見 |
| ④ 偽内部ネットワーク | 本物との区別を難しくし、探索のコストと時間を大きく引き上げる |
| ⑤ 長期相関 | 数ヶ月かけて進む潜伏型（低速APT）を、時間をまたいで検知 |

## カバレッジ — 何を守り、何を守らないか

「防御率○%」のような数字は出しません（実環境はそれほど単純でなく、誇張は信頼を損なうため）。守れる範囲と守れない範囲をそのまま示します。

**対応する**：侵入後の異常検知／データ窃取の無力化／攻撃者インフラの暴露／探索の無効化／潜伏型攻撃の検知／誤作動の抑制

**対応しない（正直に）**：
- 侵入そのものの阻止（入口防御の役割。Zeeは置き換えでなく補完）
- ネットワーク層での攻め返し（C2汚染。現時点で未実装・実証性を検討中）
- 脆弱性の修正・パッチ適用（Zeeは検知と無力化の層）

## 免責

- Zeeは現在 **概念実証（PoC）** の段階です。
- Zeeは**万能ではありません。** 侵入そのものを防ぐものでも、すべての攻撃を検知できるものでもありません。
- 機械速度の攻撃能力は進化を続けます。新世代モデルが広く使えるようになる局面では、**Zee自体の脆弱性チェックとパッチ適用が必要**になります。
- Zeeは入口防御・修正運用を**置き換えるものではなく、補完する一層**です。
